# Creator Workflow

This workflow shows how a creator can use StyleOS Protocol from intake to rule improvement.

中文说明：
这是博主完整工作流，适合造型、穿搭、形象、发型、妆造和摄影服务团队参考。

## 1. Collect Fan Info

Use [Fan Intake Form](fan-intake-form.md) to collect structured information.

Collect only what is needed for the service. Keep real photos and private materials outside this public repository.

## 2. Tag the Fan Profile

Translate intake information into StyleOS tags.

Use:

- face shape tags;
- facial proportion tags;
- facial feature weight tags;
- hairstyle tags;
- color direction tags;
- outfit direction tags;
- makeup direction tags;
- scenario tags;
- creator-specific tags when needed.

## 3. Select Rule Cards

Choose relevant rule cards based on:

- target scenario;
- goals;
- constraints;
- budget;
- maintenance level;
- creator judgment.

## 4. Generate Lite Report

Use [Report Lite Template](report-lite-template.md) to draft:

- summary;
- style keywords;
- diagnosis tags;
- recommendations;
- avoid list;
- creator notes;
- execution card;
- next action checklist.

## 5. Review Manually

Creators should review:

- whether the tags are reasonable;
- whether recommendations match constraints;
- whether the report is respectful and practical;
- whether the confidence level is clear;
- whether private data has been removed.

## 6. Deliver to Fan

Deliver the report through the creator's own service channel.

Make clear:

- what is included;
- what is not included;
- whether follow-up review is available;
- how the fan can execute the first step.

## 7. Collect Feedback

Collect structured feedback:

- Was the advice clear?
- Was it actionable?
- Did it fit the scenario?
- What did the fan actually try?
- What should be improved?

## 8. Abstract Rule Improvements

Turn repeated observations into abstract Rule Cards.

Before submitting publicly:

- remove private data;
- convert real cases into synthetic examples;
- avoid claiming certainty from one case;
- mark confidence level and evidence type.
