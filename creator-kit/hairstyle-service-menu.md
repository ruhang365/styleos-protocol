# Hairstyle Service Menu

Version: 0.2

This menu gives creators a service packaging reference for hairstyle-related StyleOS outputs.

中文说明：
这是发型服务菜单模板，帮助创作者把标签、规则、案例和理发师沟通卡组合成可交付服务。服务承诺应保持清晰、克制、可执行。

## Service 1: Hairstyle Lite Direction

- Input: intake form + current hair description
- Output:
  - primary hairstyle direction
  - 3-5 relevant tags
  - 2-3 rule card references
  - low-risk next step
- Suitable for: early exploration, low-risk change
- Not included: salon technique, exact color formula, product brand recommendation

## Service 2: Barber Brief Builder

- Input: hairstyle intake + preferred direction
- Output:
  - short barber brief
  - detailed barber brief
  - keep / adjust / avoid fields
  - maintenance note
- Suitable for: users preparing for a haircut
- Not included: professional salon assessment

## Service 3: Pre-Shoot Hair Check

- Input: shoot date, camera context, lighting notes, maintenance tolerance
- Output:
  - camera stability review
  - bangs / side hair shadow check
  - conservative haircut boundary
  - emergency avoid list
- Suitable for: profile photos, creator content, livestream setup
- Not included: on-set styling service

## Service 4: Creator Hair Memory Cue

- Input: creator positioning, scenario, existing hair habits, maintenance
- Output:
  - one controlled memory cue
  - quiet supporting elements
  - barber brief delta
  - content-format check
- Suitable for: personal IP refresh
- Not included: brand strategy package or audience research

## Pricing Notes Placeholder

Creators can add pricing privately. Do not commit real client names, payments, private links, or service records to this repository.

## Delivery Boundaries

- Label all recommendations as starter / unverified when using seed rules.
- Explain tradeoffs and maintenance.
- Offer multiple directions when input supports more than one path.
- Keep medical, invasive, body-change, and brand-product guidance outside this protocol.
