# Hairstyle Taxonomy

Hairstyle taxonomy organizes the signals and outputs used by the hairstyle module.

中文说明：
发型 taxonomy 用来把脸型倾向、头发属性、目标、限制和执行建议拆成结构化字段，方便创作者解释，也方便小 B 工具调用。

## Signal Groups

### Face Shape

See [Face Shape Tags](face-shape-tags.md).

Used to describe visual contour tendencies, not identity labels.

### Hair Attributes

See [Hair Attribute Tags](hair-attribute-tags.md).

Used to describe hair volume, texture, shape, crown height, hairline visibility, forehead exposure, maintenance willingness, and risk tolerance.

### Styling Goals

See [Hairstyle Goal Tags](hairstyle-goal-tags.md).

Used to describe what the user wants to change, such as looking cleaner, softer, more professional, sharper, or more balanced.

### Constraints

Common constraints:

- keep current length;
- avoid high maintenance;
- workplace rules;
- event deadline;
- budget;
- salon availability;
- comfort with visible change.

## Output Groups

- length range;
- layer placement;
- parting direction;
- bangs direction;
- crown volume;
- face-framing;
- weight removal;
- curl or wave direction;
- maintenance notes;
- barber brief notes.

## Pro Library Boundary

The open taxonomy is a starter structure.

Advanced expert-reviewed hairstyle mappings, real authorized case libraries, and Pro model patterns belong to future commercial or separately licensed layers unless explicitly released.
