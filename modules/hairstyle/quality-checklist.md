# Hairstyle Quality Checklist

Use this checklist before publishing or delivering a hairstyle report.

- [ ] Is there a clear user goal?
- [ ] Are target scenarios stated?
- [ ] Are face-shape and proportion tags sufficient?
- [ ] Are hair attributes included?
- [ ] Are constraints included?
- [ ] Is maintenance cost considered?
- [ ] Is risk tolerance considered?
- [ ] Does the report avoid absolute claims?
- [ ] Does the report avoid appearance-shaming and body-shaming language?
- [ ] Is there an avoid list?
- [ ] Is there barber-executable language?
- [ ] Does the barber brief include keep / adjust / avoid?
- [ ] Does the report avoid real personal data?
- [ ] Are all public examples synthetic or authorized and anonymized?
- [ ] Are rules marked starter / unverified?
- [ ] Is Pro or expert-reviewed content kept outside the open layer unless explicitly released?
