# Hairstyle Creator Guide

This guide is for hairstyle, styling, and image creators using StyleOS Hairstyle Module v0.2.

## Collect User Information

Use [Hairstyle Intake Form](../../creator-kit/hairstyle-intake-form.md) to collect goals, current hairstyle, scenario, constraints, maintenance willingness, risk tolerance, and consent state.

Do not submit raw photos or private user data to the public repository.

## Avoid Over-Promising

Use starter / unverified language. Avoid claims that imply expert certification, scientific certainty, or guaranteed result.

## Select Rule Cards

Start from:

- user goal;
- face-shape and proportion tags;
- hair attributes;
- constraints;
- scenario;
- maintenance level.

Then select applicable rules from [Hairstyle Rule Index](hairstyle-rule-index.md).

## Manually Review the Report

Before delivery, review whether:

- inputs are sufficient;
- constraints are respected;
- recommendation is actionable;
- avoid list is clear;
- maintenance cost is realistic;
- barber brief uses execution language.

## Output Barber Brief

Use [Barber Brief Schema](barber-brief-schema.md) and [Barber Brief Card](../../execution-cards/barber-brief-card.md).

The brief should describe goals, keep items, adjust items, do-not-do list, and maintenance level.

## Collect Feedback

Use [Hairstyle Feedback Form](../../validation/hairstyle-feedback-form.md) to learn whether the user understood, executed, and found the recommendation useful.

## What Can Be Contributed to GitHub

- synthetic cases;
- abstracted rule improvements;
- taxonomy suggestions;
- workflow suggestions;
- fully anonymized authorized examples.

## What Should Not Be Publicly Uploaded

- raw user photos;
- real names or contacts;
- private chat logs;
- private service reports;
- unapproved before / after material;
- identifiable creator or fan data.
