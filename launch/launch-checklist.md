# Launch Checklist

## Repository

- [ ] README first screen explains StyleOS in 30 seconds.
- [ ] Commercial boundary is visible.
- [ ] Creator Studio Lite docs are linked.
- [ ] Creator Alpha workflow is linked.
- [ ] Starter rules and synthetic cases are discoverable.
- [ ] No app, package manifest, runtime service, or external dependency is added.

## Privacy

- [ ] No real user photos.
- [ ] No real names or contacts.
- [ ] No private fan records.
- [ ] No identifiable creator information.
- [ ] All cases are synthetic or fully anonymized with authorization.

## Launch Assets

- [ ] Landing page copy ready.
- [ ] GitHub Star launch plan ready.
- [ ] Creator recruitment copy ready.
- [ ] Social posts ready.
- [ ] Release notes ready.

## Creator Alpha

- [ ] Target creator segments defined.
- [ ] Alpha workflow documented.
- [ ] Consent and anonymization reminders included.
- [ ] Feedback collection structure ready.
- [ ] Rule abstraction path defined.

## Open-Core Boundary

- [ ] Open protocol is clearly separated from hosted products.
- [ ] StyleOS Pro assets are not described as open-source.
- [ ] Trademarks and certification marks remain protected.
- [ ] Small-B users are directed toward future hosted tools, not raw GitHub workflows.
