# Hairstyle Review Checklist

Version: 0.2

Use this checklist before publishing or delivering a hairstyle report built on StyleOS starter rules.

中文说明：
本清单用于检查发型模块输出是否可解释、可执行、边界清晰。当前规则为 starter / unverified，不应包装成专家认证或数据验证结论。

## 1. Evidence and Status

- [ ] Every rule reference links to a rule file.
- [ ] Rule status is visible as `starter / unverified`.
- [ ] Evidence level is visible as `E0`.
- [ ] Confidence level is `low` or `medium`.
- [ ] Uncertain inputs are marked rather than forced.

## 2. Language Safety

- [ ] No appearance shaming.
- [ ] No body or age value judgment.
- [ ] No gender stereotype.
- [ ] No fixed promise of outcome.
- [ ] No medical, invasive, or body-change guidance.
- [ ] No brand product recommendation.

## 3. Hairstyle Actionability

- [ ] Primary goal is clear.
- [ ] Maintenance level is clear.
- [ ] Length boundary is clear.
- [ ] Layer / volume / bangs / parting notes are concrete.
- [ ] Barber brief includes `keep`, `adjust`, and `avoid`.
- [ ] Multiple viable directions include tradeoffs.

## 4. Privacy

- [ ] No identifiable photos.
- [ ] No real names, contacts, accounts, or private case details.
- [ ] No unauthorized public figure or blogger reference.
- [ ] Public examples are synthetic.

## 5. Open-Core Boundary

- [ ] Public content stays in starter protocol scope.
- [ ] Paid-service claims are not embedded into open rules.
- [ ] Pro-only logic is not mixed into seed rule files.
