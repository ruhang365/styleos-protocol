# Roadmap

StyleOS Protocol evolves from a documentation-first open standard toward stable schemas and creator-ready workflows.

## v0.1 - Protocol Skeleton

- Establish repository structure.
- Define open-core boundary.
- Publish first drafts of input schema, style tag schema, rule card schema, lite report schema, and evaluation framework.
- Provide creator kit templates and synthetic examples.

## v0.2 - Hairstyle Module

- Expand hairstyle tags.
- Add hairstyle-specific rule cards.
- Define hairstyle input and recommendation fields.
- Add synthetic hairstyle examples.

## v0.3 - Color Module

- Expand color direction tags.
- Define color preference and constraint fields.
- Add color matching rule cards.
- Add synthetic color examples.

## v0.4 - Outfit Module

- Expand outfit direction tags.
- Define outfit scenario and wardrobe constraint fields.
- Add outfit rule cards.
- Add synthetic outfit examples.

## v0.5 - Makeup Module

- Expand makeup direction tags.
- Define makeup scenario and preference fields.
- Add makeup rule cards.
- Add synthetic makeup examples.

## v0.6 - Creator Workflow

- Improve creator intake, tagging, review, delivery, and feedback workflow.
- Add more service menu patterns.
- Add community workflow examples.

## v0.7 - Report Generation API Draft

- Draft report generation API concepts.
- Define input and output compatibility rules.
- Keep the draft protocol-level only; no hosted service implementation is included.

## v1.0 - Stable StyleOS Protocol

- Stabilize schemas.
- Stabilize contribution rules.
- Stabilize evaluation criteria.
- Publish compatibility guidance for tools and creator workflows.
