# Data Flywheel

StyleOS data flywheel connects open protocol, creator usage, anonymized feedback, expert review, and better hosted tools.

## Flywheel

```text
community rules
  -> creator usage
  -> anonymized feedback
  -> expert review
  -> Pro library
  -> better hosted tools
  -> stronger standard
```

## Community Rules

The public repository starts with starter rules and community contributions.

Rules should include applicable conditions, limitations, evidence level, review status, and privacy notes.

## Creator Usage

Creators use the protocol in private service workflows or future hosted tools.

Usage should be manually reviewed by creators and should not be treated as automated expert judgment.

## Anonymized Feedback

Feedback can include clarity, actionability, creator usefulness, and report outcome after identifiers are removed.

Consent and anonymization are required before any case reuse.

## Expert Review

Experienced practitioners may review repeated patterns and help identify safer, clearer, higher-confidence rules.

Expert review should not create absolute claims.

## Pro Library

Higher-confidence and commercially valuable patterns may enter StyleOS Pro under separate authorization and commercial terms.

## Better Hosted Tools

Creator Studio can use reviewed patterns to improve intake, tagging, report generation, feedback collection, and execution cards.

## Stronger Standard

Public summaries, improved schemas, and safer rule formats can return to the open protocol when appropriate.

## Compliance Reminder

No raw personal photos in public repository.

Consent and anonymization required before any case reuse.

中文说明：
公开仓库不接收原始个人照片、隐私聊天记录或未经授权案例。任何真实案例复用都必须经过授权、脱敏和分层处理。
