# FAQ

## 1. Is StyleOS a color test?

No. StyleOS is not a consumer color test. It is an open protocol for AI personal styling workflows.

中文说明：
StyleOS 不只是色彩测试，也不是单点测试工具。它关注从信息采集、标签、规则卡、报告到执行卡的完整造型服务流程。

## 2. Is this a consumer app?

Not currently. This repository is protocol + creator tools + future hosted product direction.

Ordinary fans are expected to use StyleOS indirectly through creators, studios, small businesses, or future hosted products.

## 3. Can small businesses use this directly?

GitHub provides standards and reference materials. Small businesses can learn from the documents, but future daily usage is expected to happen through hosted Creator Studio / Cloud tools.

## 4. Is the hairstyle module expert-certified?

No. The hairstyle module is currently `starter / unverified` content for protocol demonstration and community co-creation.

It should not be presented as expert-certified, scientifically proven, or data-validated.

## 5. Can I submit real user photos?

No. Do not submit real user photos to the public repository.

Do not submit real names, contacts, private service records, unauthorized cases, or identifiable details.

## 6. Can I fork this repo?

Yes. You can fork the protocol under the repository license.

You cannot use ruhang365 / StyleOS trademarks, certification marks, logos, commercial assets, hosted services, Pro libraries, or brand assets without permission.

## 7. What is StyleOS Pro?

StyleOS Pro is a future commercial advanced layer. It may include expert-reviewed rules, validated cases, advanced models, merchant tools, certification workflows, and hosted services.

StyleOS Pro is not included in this open-source repository unless specific parts are separately released.

## 8. Why use GitHub?

GitHub is useful for open standards, community collaboration, version history, contribution review, and industry visibility.

It is not the main entry point for ordinary fans. GitHub is the protocol layer; creators and small businesses are expected to use future hosted tools for daily work.

## 9. Can creators use StyleOS now?

Yes, creators can use the open templates to design intake forms, lite reports, service menus, and barber briefs.

Current public materials are starter references, not a finished SaaS product.

## 10. Can developers build compatible tools?

Yes. Developers can build tools that read StyleOS schemas, use rule-card formats, generate draft reports, or help creators review workflows.

Compatible tools should preserve privacy, evidence levels, rule status, and open-core boundary.
