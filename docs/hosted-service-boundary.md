# Hosted Service Boundary

This document explains the boundary between the GitHub repository and future hosted StyleOS products.

## GitHub Repo is Protocol

The GitHub repository includes:

- StyleOS Protocol;
- schemas;
- creator kit templates;
- starter rules;
- synthetic cases;
- validation vocabulary;
- execution cards;
- launch and strategy documents.

The GitHub repository is for standards, interoperability, community building, and public documentation.

## Creator Studio is Hosted Product

StyleOS Creator Studio is a future hosted product for creators and small-B service providers.

Hosted product capabilities may include:

- authentication;
- dashboard;
- fan intake form;
- profile management;
- report generator;
- PDF or shareable report export;
- creator storefront;
- payment support;
- feedback collection;
- Pro library access;
- certification workflow;
- partner execution cards.

## Hosted Product is Commercial

The hosted product is commercial and is not included under the Apache-2.0 repository license.

## Small-B User Boundary

Small businesses and creators are expected to use future hosted StyleOS products rather than GitHub directly.

GitHub is useful for standards, transparency, contribution, and developer interoperability. Hosted products are how ruhang365 can provide ready-to-use tools for small-B users.

## Data Boundary

Hosted products may process private service data under separate privacy terms.

The public repository should only contain synthetic examples, fully anonymized authorized cases, or abstracted rules with no personal data.

Data generated from hosted small-B tools must not automatically enter the public repository or Pro library unless proper consent, anonymization, and authorization are in place.
