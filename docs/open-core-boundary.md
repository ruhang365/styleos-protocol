# Open-Core Boundary

StyleOS Protocol separates the open protocol layer from the commercial product layer.

## Open Layer

The open layer includes:

- protocol documents;
- input schema;
- style tag schema;
- rule card schema;
- lite report schema;
- evaluation framework;
- creator kit templates;
- synthetic examples;
- community contribution templates.

中文说明：
开源层负责标准传播，帮助社区理解和复用 StyleOS 的基本协议、字段、标签、规则卡、轻量报告和创作者工作流。

## Commercial Layer

The commercial layer may include:

- StyleOS Cloud;
- StyleOS Pro;
- Creator Studio;
- Expert Model Library;
- commercial styling knowledge base;
- verified case database;
- certified partner system;
- advanced model and report workflows;
- analytics, operations, and data feedback loops.

中文说明：
商业层负责完整线上产品、高级库、专家知识库、认证体系、真实授权案例库和数据闭环。这些能力不属于本仓库开源范围。

## Compatibility Language

Allowed:

- "compatible with StyleOS Protocol"
- "uses StyleOS Protocol schemas"
- "built using StyleOS Protocol open templates"

Not allowed without permission:

- "official StyleOS product"
- "StyleOS Certified"
- "official ruhang365 partner"
- "StyleOS Pro provider"
- any wording that implies certification, endorsement, or official partnership.

## Practical Rule

If a file helps the community understand or implement the open protocol, it belongs here.

If a file contains hosted product logic, commercial model assets, expert libraries, verified real cases, certification workflows, or brand assets, it belongs outside this repository unless explicitly released by ruhang365.
