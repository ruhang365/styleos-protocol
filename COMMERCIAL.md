# Commercial Boundary

StyleOS Protocol follows an open-core model.

The open-source repository provides the protocol, schemas, lite templates, creator kit, synthetic examples, and community reference materials.

The commercial layer may include hosted products, advanced model libraries, expert knowledge bases, certification systems, creator tools, analytics, and managed workflows.

## Open Source Layer

The following materials can be used under Apache-2.0:

- StyleOS Protocol documents
- input schema
- style tag schema
- rule card schema
- lite report schema
- evaluation framework
- creator kit templates
- synthetic examples
- GitHub contribution templates

中文说明：
开源仓库可以自由使用协议、Schema、模板、Creator Kit 和 synthetic examples，但使用时仍需遵守 Apache-2.0、数据政策、贡献条款和商标规则。

## Commercial Layer

The following are commercial services or commercial assets of ruhang365:

- StyleOS Cloud
- StyleOS Pro
- Creator Studio
- Certified Partner system
- Expert Model Library
- commercial styling knowledge base
- verified case database
- advanced model and report generation workflows
- trademarks, logos, certification marks, and brand assets

中文说明：
StyleOS Cloud、StyleOS Pro、Creator Studio、Certified Partner、Expert Model Library 等属于商业服务或商业资产，不因本仓库采用 Apache-2.0 而自动开放。

## For Creators and Small Businesses

Creators and small beauty / styling businesses may use the open protocol to design compatible workflows, intake forms, rule cards, and lite reports.

Future ruhang365 online products may provide complete hosted capabilities, advanced libraries, creator operations tools, certification, and commercial delivery workflows.

中文说明：
小 B 和创作者可以先基于本仓库搭建轻量服务流程；完整线上服务、高级库、认证和数据闭环能力可通过 ruhang365 未来提供的商业产品获得。

## GitHub Protocol vs Hosted Products

Small businesses and creators are expected to use future hosted StyleOS products rather than GitHub directly.

GitHub protocol is for standards, interoperability, transparency, and community building.

Commercial hosted products are how ruhang365 provides ready-to-use tools for small-B users, such as creator dashboards, fan intake, report generation, branded output, feedback collection, Pro library access, certification workflows, and partner execution cards.

中文说明：
GitHub 仓库面向标准、互操作和社区共建；小 B 和创作者的日常使用场景更适合未来托管产品，例如 StyleOS Creator Studio / StyleOS Cloud。托管产品是商业服务，不属于 Apache-2.0 开源仓库授权范围。

## Creator Studio Lite Boundary

The open repository may document the Creator Studio Lite direction, workflow, and feature specification.

The hosted product itself may include authentication, dashboard, report generator, creator storefront, payment support, Pro library access, and certification workflow. These are commercial capabilities unless explicitly released under a separate open-source license.

## Release and Alpha Boundary

The repository may include release copy, Star guides, Alpha recruitment pages, community onboarding, and launch calendars.

These documents are public communication and workflow materials. They do not grant access to:

- hosted Creator Studio;
- StyleOS Cloud;
- StyleOS Pro;
- expert-reviewed libraries;
- verified case databases;
- certification systems;
- commercial dashboards;
- partner execution networks.

Creator Alpha feedback may inform future commercial products, but public repository participation does not create commercial rights, certification, partnership status, or access to private systems.

## No Implied Commercial Rights

Using this repository does not grant rights to:

- present a product as an official StyleOS product;
- claim StyleOS certification;
- use ruhang365 or StyleOS trademarks as product branding;
- access commercial model libraries or expert knowledge bases;
- access hosted StyleOS Cloud or StyleOS Pro services.

For commercial partnership, certification, or advanced platform access, a separate agreement is required.
