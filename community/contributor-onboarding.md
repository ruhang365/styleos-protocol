# Contributor Onboarding

Welcome to StyleOS Protocol.

中文说明：
StyleOS 是入行365发起的 AI personal styling workflow 开放协议。贡献者可以帮助完善 schema、rule cards、synthetic cases、creator workflows 和评估标准。

## How to Understand StyleOS

StyleOS is not a consumer app. It is a protocol layer for personal styling workflows.

Core objects:

- input schema
- style tags
- rule cards
- lite reports
- execution cards
- validation and feedback
- open-core boundary

## What You Can Contribute

- starter rule cards
- synthetic cases
- schema suggestions
- report templates
- creator workflow examples
- evaluation checklist improvements
- documentation clarity improvements

## What Not to Contribute

- real user photos
- real names or contacts
- private service notes
- unauthorized cases
- exact addresses or payment records
- scraped third-party content
- product code, package manifests, frameworks, or external dependencies
- Pro-only libraries, expert model assets, verified case databases, or certification claims

## How to Submit a Rule Card

1. Read [Rule Card Schema](../docs/rule-card-schema.md).
2. Check [Rule Card Submission Guide](rule-card-submission-guide.md).
3. Use synthetic examples.
4. Set status and evidence level clearly.
5. Include limitations and privacy note.
6. Submit via GitHub Issue or pull request.

## How to Submit a Synthetic Case

1. Use a fictional profile.
2. Include scenario, input tags, selected rules, lite recommendation, execution card, feedback placeholder, and privacy note.
3. Do not include real photos, identities, contacts, or private records.
4. Link to the rule cards used.

## How to Submit a Schema Suggestion

1. Describe the current limitation.
2. Propose the field, tag, or object.
3. Explain compatibility impact.
4. Include a synthetic example.
5. Clarify whether the suggestion belongs to open protocol or future hosted product.
